# R0 · 问题框：路径锁（Path Lock）

## 1. 对内定义（已定）

```text
路径锁 Path Lock
  = 高价值「想消但被几何挡住」的牌（现：L1 组顶）
  + 全场存在 ≥1 条可规划获得的同 match-key（同点同色）路径
  + 玩家须先打开压住它的上层 / 侧向，或规划抽序

不是：
  - 唯一实体钥匙
  - 特殊点击规则 / 状态机解锁
```

| 层 | 含义 | 当前实现 |
|----|------|----------|
| L-geo | 谁压住锁、旁路多少 | 满槽 isFree 几何 |
| L-match | 同色同点有几张、在哪 | 上色+偶数；**密度未控（常 4–12）** |
| L-tempo | 何时遇到锁 | L2 易 → L1 锁难 |

## 2. 路径锁 vs 唯一钥匙

| | 路径锁（我们） | 唯一钥匙 |
|--|----------------|----------|
| 开锁条件 | 任意同 match-key free 配对 | 指定 key 实体 |
| 钥匙数量 | ≥2 偶数（可多） | 通常 1 |
| 失败感来源 | 路径没挖对 / 颜色错 / 抽错 | 找不到那一把 |
| 怒点 | 钥匙太多 → **感觉没锁** | 钥匙被埋死 → 硬骗 |

## 3. 本项目「好卡点 / 烂卡点」（设计直觉）

### 好卡点（要强化）

- 看见 L1 瓶颈 rank，但上面还有 1～2 层要先拆  
- 钥匙在 L2 挖开链末端，想通「先打桌上链再回头」  
- 同点异色在 free 上晃，逼你认清 D22  
- chain 锁：开锁 1 才露出锁 2 的钥匙位  

### 烂卡点（要消灭）

- 锁 match-key 全场 8～12 张 → 随便消总能撞上  
- 锁一 free，桌上已有 3 张同 key free → 无决策  
- 钥匙只在 stock 深层且无提示 → 纯赌抽  
- 贪心可清失败 + fallback 发局 → 偶发无解感
