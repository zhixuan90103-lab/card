# 参数卡 · Juice 剂量与动画锁输入

- 品类：休闲/解谜/动作反馈原则
- 输入：常用操作 vs 稀有完成
- 可信度：**中**（社区 + 内部 handfeel/sorting 交叉）
- 源：
  - r/gamedesign「juice」讨论：过量震屏/hitstop 疲劳
  - `research/handfeel/00` · sorting `05` · `03` 体验合同
  - UX StackExchange / Material：小动 150–200、大动 300–400

| 项 | 观察值 | 备注 |
|----|--------|------|
| 剂量 | 常用动作 **薄**；稀有事件 **厚** | 消一对薄于清桌 |
| 震屏 | 常被要求可关；精确扫视场景慎用 | 本产品 **默认关** |
| 锁输入 | 长动画锁死 → 玩家怒（Parking skip 帖等） | 消牌可 busy；勿叠长庆祝 |
| 选中 | 即时 | ≤100ms |
| 消牌 | 短清脆 | 200–300ms；连消可跳过/并行待定 |
| 非法 | 秒懂、不重罚 | 改选中 / 轻抖可选 |
| 过关 | **短结算，尽快可点下一关** | sorting 05：≤1s |
| free 高亮 | 层叠消常见「可点高亮」 | 本产品 = 亮面本身 |
| HUD | 失败浮层要 **主 CTA 可达** | 撤销/重开/新局 |

**可迁移：**  
- E03 中等 juice；E12 略厚但仍短  
- `isBusy` 仅绑消牌飞走；翻面尽量不锁  
- 非法 **禁止** 强震屏当惩罚  

**不可迁移：**  
- ARPG hitstop / 大震屏 combo  
- 三消全屏 cascade 粒子  

**对本产品决策建议：**  
- 消牌中 **维持全锁**（实现简单、防双点乱序）若体感拖，再改「缓冲下一选」  
- 默认 **不**做 haptic（Web 支持差）；音效 P2
