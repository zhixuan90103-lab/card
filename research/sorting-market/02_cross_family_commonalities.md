# ② 跨家族真共性 v2（三轮检索后 · 可证伪）

> **声称范围收窄：** 下列 H1–H10 的「跨家族」主证来自 **A / C / E(Screw) / R1** 机制千关支。  
> **B-cozy / F(Perfect Tidy) / Unpacking** 只作边界对照，不强制共享「参数化千关」假设。  
> 详见 `08_round_synthesis_final.md` 的 H11–H13。

每条格式：**假设 → 机制含义 → 证据 → 反例/边界 → 设计含义**

---

### H1 目标态一眼可读（E11）

- **假设：** 成功态几乎不靠长文案（同色满管 / 车全离场 / 红块出门）。  
- **证据：** Water Sort 商店与攻略规则极短；Parking Jam wiki「按序移出」；Unblock「放走目标块」。  
- **反例：** 强叙事整理模拟需要情境目标。  
- **设计：** 过关判定可视化 > 文字任务列表。

### H2 难度来自约束加密，不来自规则膨胀（支撑千关）

- **假设：** 后期仍是同一套操作语言，变的是数量、层序、空位、挡路深度。  
- **证据：** Water 高关（含 3000+）仍谈 empty tubes；Screw 五百关+同玩法；Parking/Bus 长尾关号。  
- **反例 / 附条件：** 市场常插入 **extra tube 墙、booster 墙、假死局**（indie 开发者 rage-quit 商店头部）——规则语言未变，但 **可解性信任** 被破坏。  
- **设计：** 参数加密 ✅；人工「不买过不去」❌（见 H12）。

### H3 主认知核稳定且 ≤2 个

| 家族 | 主核 | 玩家误认 |
|------|------|----------|
| A | C2 空位 + C3 堆叠 | 「在倒水/整理颜色」 |
| C | C1 反推 + C7 阻塞搜索 | 「在停车/开车」 |
| E Screw | C7 拆除阻塞 + C5 分色入孔 | 「在拧螺丝」 |
| R1 Goods | 消除 + 槽压（非纯 B） | 「在整理货架」 |
| B-cozy / F（边界） | C5 精确摆放 / C8 状态削减 | 「在整理房间」— 关卡逻辑不同 |

- **设计：** 教程教 **核**（空管是工作区；先想谁挡谁），主题只负责秒懂。

### H4 空位 / 工作区是 A 系灵魂资源

- **假设：** A 的策略核心不是「认识颜色」，而是 **保护与使用 empty tube**。  
- **证据：** 多篇 strategy：create/protect empty tubes；勿过早填死最后空管；部分关必须 extra tube。  
- **设计：** 难度曲线可直接调「有效空位数」与「层序交错度」。

### H5 低重开成本 / 安全探索（E9）支撑「再来一关」

- **假设：** 无罚时、可重试时，近失转化为重开而非弃游。  
- **证据（R3 强化）：** 多帖标题即「sorting without timer」；Zen sorting / Cozy 求无钟；Screw 求 no time limit；开发者新作以 no-timer 为卖点。  
- **证伪尝试：** 未找到「整理类主流玩家热爱 timer」的稳定话语；timer 几乎只带来焦虑叙事。  
- **反例受众：** 竞速/冲榜另一细分，非本调研主用户。  
- **设计：** **默认无倒计时**（非可选项级，是默认级）；失败秒开；撤销优先。

### H6 双级反馈：局部完成 + 全局完成

- **假设：** 单次「倒满一色 / 一辆出库 / 一组三消」给小高潮；整关清台给大闭合。  
- **证据：** Water 单色完成；Parking 单车驶离；R1 三消爆发（多巴胺被社区点名）。  
- **设计：** 动画与音效预算优先砸在「局部完成」瞬间。

### H7 近失可设计（E3）

- **假设：** 空位将尽、只剩堵车、差一组清台 → 牵引继续。  
- **证据：** 玩家 stuck on high level 仍回坑；addicted 表述在 Goods/Water 均出现。  
- **风险：** 近失 + 不可读死局 + 广告 = 怒卸。  
- **设计：** 死局应可被玩家归因（「我堵死了」）而非怀疑关卡作弊。

### H8 脚本可弱，核必须硬（假皮可爆）

- **假设：** A 几乎无生活真实感仍可全球级传播；C 用车皮加速理解但不改变图序规划核。  
- **证据：** Water/Ball 抽象符号；Unblock 无车仍同构 C；Parking 主题=车但操作≠真实驾驶。  
- **设计：** 主题选型服务获客与可读；留存抓 C/E。

### H9 单局节奏以碎片为主

- **假设：** 30s～3min 为甜区；可边听书/睡前/产假「无聊游戏」场景。  
- **证据：** Reddit：maternity leave mindless、winding down for bed、audiobook background、obsessive short sessions。  
- **设计：** 单关可中断、可秒开；深度靠关卡链不靠单关史诗。

### H10 体验最大外部杀手：打断与强制压力（非核本身）

- **假设：** 社区对 Goods/Closet 的爱与恨常同时存在：爱 sort，恨 ads/timer。  
- **证据：** 「completely addicted but can't stand ads」；airplane mode 规避；pay for no ads 讨论；StopGaming 有人两周上瘾后卸载。  
- **设计含义（体验侧）：** 核心循环本身可极粘；**打断会切断 E7 节奏催眠与 E9 安全感**。本调研不展开变现，但记为体验约束。

---

### H11 约束规划 / 消除清台 / 精确摆放 三支逻辑（三轮新增 · 自洽修订）

- **假设：** 「整理欲」口头下至少三套产品逻辑，关卡生产与上瘾结构不同。  
- **证据：** ALTTL/PT 要 exact placement；Water/Screw/Jam 要空位/反推与长关号；R1 要 match-3 清台（可千关但非归位核）。  
- **设计：** 立项先选支；**禁止把 R1 与 A/C 共用「整理 DNA」表述**。

### H12 可解性信任是基础设施（三轮新增）

- **假设：** 玩家接受难，不接受「被关卡/道具欺骗」。  
- **证据：** impossible without ads；fake unsolvable；extra tube paywall 比较帖。  
- **设计：** 生成器可解证明 + 透明规则。

### H13 脚本需求真、千关供给在抽象核（三轮新增）

- **假设：** 社区要 shelving/organize，头部可规模化供给却是 A/C/E 与 R1。  
- **证据：** pure organize 帖常落到 Unpacking/cozy；千关话语在 Water/Screw/Jam。  
- **OP-1（待验，非结论）：** 半真脚本 + 硬核参数化 + **非三消纯归位** 是否成立，需原型验证（见 `09` §3.2）。

### H5 证据边界（自洽附注）

- 检索场偏向 `no timer` / 反 timer 话语；**未系统抽样竞速向用户**。  
- 自洽表述：在 **放松·整理·碎片** 场，无 timer 是强共识；不作全品类数学证明。

## 共性优先级（做产品时的「非谈判项」候选）

1. 规则语言极短 + 目标态可视  
2. 单一稳定认知核 + 参数化加难  
3. **默认可解 + 无强制道具墙（H12）**  
4. **默认无 timer（H5）** — 适用范围：碎片放松向机制 puzzle  
5. 局部完成反馈密集  
6. 重开/撤销成本极低  
7. 主题与核的关系写清楚（真/半/假）  
8. 立项先选 H11 三支之一，禁止混用方法论

## 证伪计划（后续实机）

| 假设 | 怎么证伪 |
|------|----------|
| H2 | 若头部产品后期频繁引入全新操作词 → 削弱 |
| H5 | 若无 timer 版本留存显著更差 → 重估 |
| H8 | 若抽象版全面败给强生活皮同核 → 重估脚本权重 |
