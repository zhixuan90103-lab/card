## 2. 共性假设修订（H 状态）

| H | 原判断 | 三轮后 | 修订 |
|---|--------|--------|------|
| H1 目标可视 | 强 | **维持** | cozy 精确摆放有时「已摆好仍不过」→ 目标态若隐式会伤体验 |
| H2 约束加密不换规则 | 强 | **附条件维持** | 千关成立；但市场常插入 **extra tube / booster / 假死局** 破坏「纯约束」信任 |
| H3 主核≤2 | 强 | **维持** | Screw= C7+C5；Water=C2+C3；Jam=C1+C7 |
| H4 空位灵魂（A） | 强 | **维持** | 高关仍谈 empty tubes；3145 等 |
| H5 无 timer / 低惩罚 | 强 | **强化 ★★★** | R3 全是「求 no timer」；未见「爱 timer 的主流整理党」 |
| H6 双级反馈 | 强 | **维持** | Screw 拧下/入孔；PT 局部收齐；Water 满色 |
| H7 近失 | 中 | **维持+风险** | stuck days；与 booster 墙叠加则转怒 |
| H8 假皮可爆 | 强 | **维持** | Screw/Water 假或半真均爆 |
| H9 碎片节奏 | 强 | **维持** | PT nighttime；Water mindless |
| H10 广告打断 | 强 | **强化** | Screw watch-ads-or-lose；PT 买去广告；Goods 同构 |

### 新增 H11–H13

**H11 · 三支产品逻辑（自洽修订：R1 不再挂进「空位规划」）**  
- **约束规划千关**（A / C / E-Screw）：参数加密 → 易千关  
- **消除清台千关**（R1）：可千关，核是消除，**非整理归位 DNA**  
- **精确摆放舒适**（ALTTL/Unpacking/PT）：手作正解 → 难无限工厂  
- **不可用同一套关卡方法论硬套**

**H12 · 「可解性信任」是体验基础设施**  
- 玩家容忍难，不容忍「不看广告/不买管就过不去」或 bug 关  
- 设计：可解验证 + 透明规则 > 人工难度尖刺

**H13 · 生活脚本需求真实，但千关供给在抽象核**  
- 玩家会找 shelving / organize / tidy（B 需求）  
- 供给端能规模化的是 Water/Screw/Jam（及 R1 三消）  
- **OP-1 待验假设（非已证机会）：** 半真脚本 + 参数化硬核 + 非三消纯归位 — 需原型验证

## 6. 对「机制向千关」产品的决策地图

```text
                    生活脚本强度
                 低 ────────── 高
              ┌─────────┬─────────┐
     易千关   │ A Water │ C Parking│  ← 机制向主推荐
     约束规划 │ E Screw │ (半真)   │
              ├─────────┼─────────┤
     易千关   │ R1 三消 │          │  ← 参照：可规模但非整理核
     消除清台 │ (假皮)  │          │
              ├─────────┼─────────┤
     难千关   │ (少)    │ B ALTTL │  ← 体验参考，关卡生产不同
     手作关   │         │ F PT    │
              │         │ Unpacking│
              └─────────┴─────────┘
```

**若目标=成百上千关机制向：** 优先 **A / C / E(Screw 结构)**；用半真皮提升可读；默认无 timer；死保可解性信任。  
**若目标=整理欲+舒适：** 学 **PT / ALTTL** 的脚本与手感，但别承诺同一套关卡工厂 ROI。  
**OP-1 待验：** 高脚本 + 可参数加密的 **纯归位（非三消）** —— 社区有需求话术，头部供给几乎空；**不等于已验证机会**。
