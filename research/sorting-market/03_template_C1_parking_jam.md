# ③ 样板深挖：C1 Parking Jam 系

**代表：** Parking Jam 3D 及同核「点车离场 / 移车出库」  
**方法：** 四层拆解 + 真/半/假 + 上瘾/舒适/流失  
**用途：** 全品类分析的可复制模板  

---

## 四层拆解

| 层 | 内容 |
|----|------|
| **① 主题皮** | 3D 停车场、各色车辆、出口、偶尔 NPC/警察等装饰障碍 |
| **② 现实脚本 L2** | 狭窄车位被堵、要挪车让路、交通堵塞焦虑的「情境熟悉感」 |
| **③ 认知核** | **主 C1 依赖链反推** + **C7 阻塞图搜索**；执行段 **C9**；防锁死 **C11** |
| **④ 情绪兑现 E** | E4 计划掌控、E5 顿悟、E2 疏通释放、E6 轻量征服；浅关可有 E7 节奏 |

### 真 / 半 / 假 判定：**半真半假**

| 真的部分 | 被替换的部分 |
|----------|--------------|
| 挡路关系、出口方向、空间拥挤 → 用生活经验 **读关** | 真实驾驶、视角责任、物理停车技术 |
| 「堵车好烦，疏通真爽」的情绪脚本 | 变成 **图序规划 puzzle** |

**玩家自我叙述：**「我在挪车 / 停车解困。」  
**认知核复述：**「我在算依赖顺序：谁挡谁，先动哪辆才不锁死。」

→ 上瘾解释 **优先跟 C1/C7 与 E4/E5**，不跟「模拟驾驶真实感」。

---

## 核心循环（单关）

```text
扫视出口与挡路关系
  → 选择当前可合法移动的车
  → 移开 / 出库（局部反馈）
  → 更新阻塞图
  → 重复直至清空
失败：无合法步 / 走进死局 → 重开（成本应低）
```

**与 Unblock Me（C3）对照：**

| | Parking Jam | Unblock Me |
|--|-------------|------------|
| 皮 | 车·停车场 | 木块·网格 |
| 核 | 同属 C1+C7 | 同属 C1+C7 |
| 读关 | 生活脚本加速 | 纯几何 |
| 结论 | 皮可换，核可复用 | 假皮对照成立 |

## 上瘾结构（样板）

| 阶段 | 机制 | 玩家状态 |
|------|------|----------|
| 入局 | 目标极简：车都弄出去 | E11 安心 |
| 中段 | 可动集合缩小，需反推 | E4 计划 |
| 卡点 | 差一两辆 / 看似无解 | E3 近失 |
| 击中 | 发现关键首步 | E5 顿悟 |
| 收尾 | 连续出车动画链 | E2 释放 |
| 再来 | 下一关参数加密 | 「再来一关」 |

社区侧补充：有人当 **mindless + 难度起伏** 的听书背景游；有人推到 **极高关号**（同规则长尾），支持 H2 千关。

---

## 舒适结构

- 操作原子：点/滑一辆，规则短  
- 无真实碰撞后果（商店话术甚至卖「撞了也不用赔」的释放）  
- 成功路径可视化（车沿轨道出去）  
- 可做放松向（与 hard 关并存）

---

## 流失 / 怒点（体验向）

| 点 | 说明 | 归类 |
|----|------|------|
| 重复感 | 设计讨论里有人批 repetitive | 关卡花样不足（H2 的代价） |
| 商业与进度噪音 | 广告、IAP、进度异常（社区有怨） | 体验干扰，非 C 核 |
| 死局不可读 | 若看不出为何卡住 | 伤 E9，转愤怒 |
| 3D 误点 | 透视导致点错车 | **手感问题** → 见专章 |

r/gamedesign 帖显示：认真设计分析供给不足，克隆与变现讨论更多——**说明品类方法论空白，恰是我们调研的价值。**

---

## 设计启示（若做 C 系）

1. **关卡难度单位** = 依赖链深度 × 岔路数 × 可逆空间，不是更仿真的车控。  
2. **教程** 应教「先找挡出口的车 / 倒推」，不要教开车。  
3. **主题** 可车可巴士可动物（Animal Farm 变体），核不变。  
4. **局部反馈** 出车要爽；全局清空要有收束。  
5. **与 A 差异化：** A 卖空位周转催眠；C 卖顿悟与顺序；勿做成又倒水又出车的四不像，除非有意杂交。
