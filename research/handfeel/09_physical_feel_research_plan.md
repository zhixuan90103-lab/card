## 0. 一句话目标

> 把产品方定义的 **浮动选中 · 意图高亮 · 对撞+抛物线出场 · 抽移再翻 · 洗回扣背依次回库 · 有缩放呼吸的翻牌**，用 **UI/UE 动效经验 +《动画师生存手册》原理 + 纸牌动效实践** 检索成 **可执行参数与事件表**，服务顿悟扫视，而不是做成过场电影。

**产品滤镜（强制）：**

```text
主卖 = 顿悟（消序 / 抽不抽 / 掀哪条线）
输入 = 点选 + 拖放；成功峰值同级
合法 = isFree + 同色同点（D17/D22 不改）
引擎 = Pixi 2D（伪翻 / 2D 抛物线，禁回流真 3D 选型）
剂量 = 成功可厚、日常薄、非法不重罚
连消 = 常用路径总时长可控；洗回多张必须加速策略
```

## 1. 需求钉（检索对象 · 已方拍板意向）

> 下列为 **产品意图**，检索负责 **可做参数、竞品范式、风险与反模式**，不负责改规则。

### 1.1 点选

| 阶段 | 意图 |
|------|------|
| 点中 free | **iOS 长按图标式浮动**（离层、轻浮，非仅 y-4） |
| 再点可配目标 | 两张 **向彼此对撞** |
| 对撞后 | **上抛 + 旋转 + 物理抛物线** 落到屏外 |

### 1.2 拖放

| 阶段 | 意图 |
|------|------|
| 拖中 | 被拖牌 **略放大** + 跟手 |
| 松在可消上 | 与点消 **同一套** 抛物线旋转出场（峰值同级） |
| 非法 | 仍 **弹回纠正**（检索可细化缓动，不重罚） |

### 1.3 意图识别（舒适操作）

| 阶段 | 意图 |
|------|------|
| 点中/拖起 A | 盘面 **同 match-key 且 free** 的牌在合理范围内 **轻放大** |
| 目的 | 辅助模糊瞄准；**非 free 不高亮**（防误导可点） |

### 1.4 翻牌（新 free / 抽到位后）

| 阶段 | 意图 |
|------|------|
| 背→面 | **有翻动过程** |
| 附加 | 过程中 **略放大再回原大**（模拟掀开感） |

### 1.5 抽牌区 → 抽出叠（E07 升级）

| 阶段 | 意图 |
|------|------|
| 1 | 牌 **先位移** 到抽出叠目标位 |
| 2 | **再翻开**（与 1.4 同技法家族） |

### 1.6 洗回（E08 升级）

| 阶段 | 意图 |
|------|------|
| 1 | 抽出叠牌 **完全扣回牌背** |
| 2 | **依次** 回到抽牌区落位叠放 |
| 约束 | 多张时必须有 **加速/成组** 策略，防拖死 |

## 2. 三知识支柱（检索从哪来）

| 支柱 | 内容 | 本产品映射 | 检索重点 |
|------|------|------------|----------|
| **A. UI/UE 动效经验** | 微交互时长、状态机、busy、触控拖放、意图反馈、剂量 | 选中/拖/合法/非法/引导高亮 | Material/iOS HIG 量级、DnD 状态、模糊目标 |
| **B. 动画师生存手册** | Williams *Animator's Survival Kit*（中译常称 **动画师生存手册**） | 弧线、时间、交叠、预期、次要动作 | **原理迁移**，非逐帧抄角色动画 |
| **C. 纸牌动效知识** | 发牌、翻牌、收牌、洗牌、Solitaire 触控 | 抽移、翻、扣背、回库、消出场 | 数字纸牌/接龙/炉石仅取物理语言 |

### 2.1 《动画师生存手册》→ 本产品「可迁移 / 慎用」

| 原理 | 可迁移 | 慎用/禁止 |
|------|--------|-----------|
| **Timing 时间** | 短动作快、重结果略慢；表驱动 ms | 角色表演级长镜头 |
| **Arcs 弧线** | 抛物线出场、抽移路径 | 无意义乱飘挡扫视 |
| **Anticipation 预期** | 选中上浮=将要操作 | 消前过长蓄力 |
| **Follow-through / Overlapping** | 洗回依次、多张 stagger | 无限 cascade |
| **Slow-in / Slow-out** | ease-out 落点、ease-in 离场 | 全程 linear 机械感 |
| **Secondary action** | 影、轻 scale 呼吸 | 粒子狂欢抢主动作 |
| **Exaggeration** | 对撞瞬间、翻中 scale | 牌面严重 squash 毁可读 |
| **Solid drawing / Staging** | 一张主焦点；消时对撞中心清晰 | 满屏同时抢戏 |
| **Squash & Stretch** | **极轻** 或不用 | 扑克识别靠固定轮廓 → **默认关** |

检索时每条原理必须落到：**事件 ID + 时长/缓动/路径一句**，禁止只写「更有生命力」。
