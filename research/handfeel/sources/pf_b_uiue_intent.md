# PF-B · UI/UE 动效 + 意图反馈

**日期：** 2026-07-22  
**轮次：** Round B  
**可信度：** 中–高（NN DnD + Material/iOS 离层常识 + 既有 fa_*）  
**源：**

| # | 源 | 要点 |
|---|-----|------|
| 1 | [NN/g Drag-and-Drop](https://www.nngroup.com/articles/drag-drop/) | 触屏：抓取态清晰；重排/归位约 **100ms** 级；中心重叠判定更稳 |
| 2 | [Smart Interface DnD](https://smart-interface-design-patterns.com/articles/drag-and-drop-ux/) | 态：rest / lifted / transit / success / error；抬升=影+outline+可选倾 |
| 3 | 既有 fa_a3 / fa_c2 | 拖 scale 1.04；成功峰值同级；过程可分叉 |
| 4 | iOS 图标物理感（WWDC 材质/影层描述） | **离层=影+轻微抬升**；编辑 jiggle **不抄**进对局 |
| 5 | 既有 NN 动画时长 / Appy 200–300 | 微交互 ≤100–120；标准 200–300 |

---

## 1. 选中「iOS 图标浮动」拆层（PF1）

| 层 | 推荐 | 勿做 |
|----|------|------|
| 抬升 | **y −8～−12** design px（强于旧 −4） | 抬到挡半桌 |
| Scale | **1.05～1.08** 出态 80–120ms ease-out | 1.15+ 挡邻牌 |
| 影 | 加深/略大（secondary） | 无影只 scale |
| 描边 | 保留金边 3px（状态可读） | 仅靠浮动无边 |
| z | 抬升 +1000 级 | — |

**与拖中差异：** 拖中 scale 可 **1.06～1.10** 且跟手；选中静态浮动略弱于拖，避免两态糊。

---

## 2. 意图高亮（PF4）

| 规则 | 推荐 |
|------|------|
| 触发 | 点中或拖起某 free 牌 A |
| 目标 | **isFree 且 canMatch(A,·)** 的牌 |
| 表现 | scale **1.04～1.06**；可选极弱描边；**默认无脉冲**（防噪） |
| 出态 | ≤80ms |
| 关闭 | 取消选中 / 松手非法弹回 / 消完 |
| **禁止** | 高亮非 free（破坏「盖住点不到」） |

**模糊操作：** 拖近可配目标时，目标可再 **+2～4px 抬升或金边加亮**（drop affordance），松手才播出场。

---

## 3. 对撞汇合（PF2）

| 项 | 推荐 |
|----|------|
| 路径 | 两牌中心 → **汇合中点**（可略偏上 10～20px） |
| 时长 | **100–160ms** ease-in-out |
| 结束 | 重叠瞬间 → 接 P-exit（可共享 1 帧「贴合」） |
| 点消 | 第二张点下即开汇合 |
| 拖消 | 合法松手：可从拖位与目标位汇合（或短对齐后出场） |

---

## 4. 合法 vs 非法（UE）

| 态 | 反馈 | 剂量 |
|----|------|------|
| success | 汇合 + 抛物线出场 | L3 厚 |
| error drop | snapBack 140–180；无红闪 | L2 |
| 点非 free | 静默 | L0 |
| 改选 | ≤100ms 切浮动到新牌 | L1 |

---

## 5. busy / 可跳过（PF8 预告）

| 段 | 锁输入 |
|----|--------|
| 汇合+出场 | **是**（整段 P-meet+P-exit） |
| 选中/意图高亮 | 否 |
| 抽移 | 短锁可选 |
| 抽翻 / 新 free 翻 | **否**（优先） |
| 洗回 | 建议短锁；N 大加速 |

**可跳过：** 连消时若下一点在出场结束前，可评估「完成当前出场最短化」；本检索轮只记原则，POC 定。
