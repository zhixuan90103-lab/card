## 2. 可复述结论（迁移用）

1. **软键盘触点是噪声观测**，不是理想键心点击。  
2. 用 **笔迹/触点空间模型** \(P(\text{touch}\mid\text{key})\)（放置误差）× **语言模型** \(P(\text{letter seq})\) 做解码，可 **显著降错**。  
3. 系统输出的是 **最可能字符串/词**，不是「AABB 命中哪个键就上哪个字」。  
4. 自动更正与预测是同一思想的产品壳：观测噪声 + 合法词典/LM。

**不宣称：** iOS 内部权重；「零错误」。

---

## 3. 对本项目的可迁移 / 不可迁移

| 可迁移 | 不可迁移 |
|--------|----------|
| 落点 = 噪声；用 **似然+先验** 解码意图 | 字母 LM / n-gram 词表 |
| **合法集**（词典）约束候选 | 把 canMatch 当成可学习语言模型去改规则 |
| 几何近 + 上下文合理 → 选目标 | 每键高斯参数照搬 |
| 错了可删/可恢复 | 输入法级连续纠错 UI |

**映射一句：**

```text
键 ↔ free 牌
词/合法串 ↔ 合法操作（可配 pair / tap / draw）
P(touch|key) ↔ geom(drop, card)
P(w|ctx) ↔ I[canMatch]（规则硬先验，非统计 LM）
```

---

## 4. 对 DropDecoder 的直接含义

- 候选必须先落在 **合法集**（free）。  
- **仅几何最近**不够：近处非法/不可配应输给稍远可配（先验权重大）。  
- 拒绝阈值 τ：似然过低 → 不提交（snapBack = 不「强行更正」）。
