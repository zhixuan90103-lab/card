## 观察

### 为何双路径是正统

- **点选：** 抬手看清目标、适合精密扫视（顿悟友好）  
- **拖放：** 直觉、连续、营销片好看  
- 评论共识：只拖易挡屏；只点在部分用户显「慢」→ **两者并存且结果等价**

### 「结果等价」≠「过程同动画」

| 阶段 | 可不同 | 必须同 |
|------|--------|--------|
| 选中 / 拖起 | y-4 vs scale 1.04 | 都立刻有反馈 |
| 过程 | 第二下点 vs 跟手拖 | — |
| **成功提交** | — | **同一 L3 兑现**（同 flyAway / 同时长 / 同 busy） |
| 失败 | 改选 vs 弹回 | **都不重罚** |

### 本产品实现对照

| 项 | 点选 | 拖放 | 一致？ |
|----|------|------|--------|
| 成功消 | flyAway 280 | flyAway 280 | ✅ 同函数 |
| 失败 | 改选 ≤100ms | snapBack 180 | ✅ 宽容语义不同形 |
| busy | match 中锁 | 拖中 + match 锁 | ⚠️ 拖中 isBusy 禁其它操作合理 |
| 阈值 | — | 8 design px | ✅ 已有 |

## 可迁移原则（定稿意向）

```text
1. 合法消：无论点/拖 → 同一 flyAway（E03a ≡ E03b 峰值）
2. 过程态：允许选中语言 ≠ 拖中语言
3. 失败：无惩罚色；弹回 = 纠正
4. 手势消歧：移动阈值后锁定 drag，不再当 tap（代码已做）
5. 成功反馈强度不因路径打折（禁止「拖消更炫、点消瞬删」分叉）
```

## 对本产品

| 决策 | 结论 |
|------|------|
| E03b | 松手合法 → **同 E03a flyAway**（已满足，规格写死） |
| E-drag | 抬起/跟手单独成行；不另做第二套消牌特效 |
| 禁止 | 仅为拖加粒子/震屏而点选没有 |
| 验收 | 盲测：录像点消与拖消，旁观者应说「消的感觉一样」 |
