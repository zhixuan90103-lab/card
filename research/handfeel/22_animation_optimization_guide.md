## 1. 总原则（先记住这 6 句）

1. **时间决定性格，间距决定重量**  
   同一 150ms：匀速 = 塑料滑轨；缓入缓出 = 有质量的短动作。

2. **一次只讲一件事（Staging）**  
   发牌时：讲「牌怎么落座」。  
   配对时：讲「消掉与抛出」。  
   不要同时震屏 + 满粒子 + 大数字。

3. **干净是差异化**  
   默认：**无震屏、无 CRT 晃、无盖牌粒子**。  
   高光留给「消成功 / 过关」，不要铺在每一帧 idle。

4. **飞行高、落座低（z 纪律）**  
   动中走 FX 带（draw≈8500 / 飞入≈9500）；落稳立刻回 puzzle / stock / waste 静止层。

5. **可操作优先于表演**  
   已钉：配对 **上抛开始即可操作下一张**；正在翻的那张禁拖。  
   发牌：建议全程 busy，最后一张进入 settle/flip 再解锁（可微调）。

6. **复用现有家族，不要另起炉灶**  
   发牌 ≈ 抽牌飞入的「批量版」；落座翻 ≈ P-flip；间隔 ≈ 洗回 stagger。

## 2. 优化优先级（建议排期）

| 优先级 | 主题 | 现状 | 建议动作 | 体感收益 |
|--------|------|------|----------|----------|
| **P0** | **进关谜题发牌** | 缺或偏弱 | 做短 cascade（见 §3） | 开场仪式感 + 遮挡可读 |
| **P0** | **z / 落座回层** | 已有 CARD_Z | 发牌必须接同一套 | 防穿帮 |
| **P1** | **发牌顺序 = 遮挡深度** | 未钉 | 底层先发 | 物理不撒谎 |
| **P1** | **落座 breath / 小倾角** | 配对翻已有 | 发牌翻复用 1.3 / ±3–8° | 实体感 |
| **P2** | **极短 Anticipation** | 无 | 可选库区 30–50ms 微压 | 少「瞬移感」 |
| **P2** | **Follow-through** | 部分有 | 位先到、角/扇后稳 1 拍 | 少硬贴 |
| **P3** | **结果高光 juice** | exit 已较满 | 保持；勿加到发牌 | 爽点集中 |
| **P3** | **跳过发牌** | 无 | 重开 / 二次进关加速或 skip | 老手友好 |
| — | 常驻晃动 / 强震 | 产品拒绝 | **不做默认** | 保干净 |

## 3. 进关发牌（本轮重点）

### 3.1 目标体验（一句话）

> 从库区（或屏底）**利落 cascade** 铺满谜题区：像在发一副有重量的牌，约 **1 秒内**讲清「谁压谁」，然后马上能玩。

### 3.2 推荐时序（草案参数）

| 参数 | 建议值 | 来源 |
|------|--------|------|
| 单张飞入 | **120–160ms** | 14 飞入族 · 手册「轻快标准短」 |
| 张间间隔 stagger | **40–50ms** | 洗回 gap / recPause 家族 |
| 缓动 | **ease-in-out** | 14 已钉飞入 |
| 轨迹 | **直线 · 无上抛弧** | 14 已钉 |
| 起点 | Stock 锚点（≈ y600）或屏底外 | 品类共识 |
| 终点 | 各谜题槽 layout 位 | 布局 runtime |
| 飞行 z | FX（draw **8500** 或 meet 族 **~9500**） | 14 / CARD_Z |
| 落稳 z | puzzle 静止层 | 14 |
| 可选预备 | 库区 **30–50ms** 微下沉 / scale 0.98 | 手册 Anticipation（极短） |
| 落座翻面 | 复用 flipMs **~180** · breath **1.3** · tilt **±3–8°** | 14 P-flip |
| 总时长（约 20 张） | 约 **0.9–1.2s** + 末张 flip | 控制 &lt; 1.5s 为佳 |
| 张数多时 | N&gt;16：stagger 收到 **20–25ms** 或成组 2 张 | pf_c 回库加速思路 |

### 3.3 顺序（比缓动更重要）

| 方案 | 做法 | 何时用 |
|------|------|--------|
| **A · 深度优先（推荐默认）** | 被压最多 / 底层先发，表层最后 | 层叠谜题；遮挡语义最正 |
| B · 行扫描 | 顶行→底行，行内左→右 | 更像经典 Pyramid 建塔叙事 |
| C · 列扫描 | 左列→右列 | 一般不优先 |

**原则：** 后发的牌在 z 与遮挡上应能压住先发的牌；动画结束态 = 规则态。

### 3.4 牌面策略（三选一，需产品拍板）

| 方案 | 观感 | 成本 |
|------|------|------|
| **① 正面直飞（推荐 MVP）** | 清爽、快 | 低；无翻 |
| ② 背飞 + 落座翻 | 仪式感强 | 中；注意总时长 |
| ③ 仅内层背、表层明 | 偏 TriPeaks | 中；要规则支持 |

**MVP 建议：①**；若要「发牌感」再升 ②，但 stagger 宜更紧。

### 3.5 输入锁

```text
发牌开始 → isBusy = true（禁点/拖）
最后一张：
  · 进入落座 settle，或
  · 进入 flip 阶段
  → 清全局 busy
正在飞 / 正在翻的单卡：不可拖
已落稳牌：仅在 busy 清除后可交互
```

与 14 的差异说明：配对允许「上抛即可操作」是因为 **结果已提交**；发牌时 **布局未就绪**，故建议更严 busy。

### 3.6 伪流程（实现提纲）

```text
onLevelEnter(layoutCards):
  show stock/waste frames
  isBusy = true
  optional: antic(stock, 40ms)

  order = sortByOcclusionDepth(layoutCards)  // 底层先
  for i, card in order:
    delay = i * staggerMs
    spawn at stock (or offscreen)
    tween to slot:
      duration = clamp(120–160)
      ease = easeInOut
      z = FX
      pivot = card center
    onArrive:
      z = puzzleRest
      if needFlip: flipToFace (breath 1.3, tilt rand)
      mark settled

  onLastCardSettlingOrFlipping:
    isBusy = false
```

### 3.7 发牌反模式（直接当 checklist）

- [ ] 开场 &gt;2s 还不能操作  
- [ ] 匀速 linear 无缓动  
- [ ] 大弧上抛飞入谜题  
- [ ] 飞行钻到已落座牌下面  
- [ ] 落稳仍停在 FX 层  
- [ ] 发牌中可点飞牌导致错位  
- [ ] 满粒子 / 默认震屏  
- [ ] 每张 200ms × 20 张串行无加速

## 4. 现有动画族 · 保持与微调

下列 **已在 14 钉落地**，优化原则是 **巩固一致性**，不是重做。

### 4.1 点消 / 拖消

| 项 | 现行 | 建议 |
|----|------|------|
| 点消 meet | A1→A2 直线 120–160ms，无弧 | **保持** |
| 拖消 | skipMeet，松手即 exit | **保持** |
| exit | 分抛 + 自旋 + 无淡出 | **保持**；这是主爽点 |
| 上抛始可操作 | 已钉 | **保持**（节奏核心） |
| 上抛始翻 free | 已钉 + breath 1.3 | **保持** |

**微调空间（低优）：**  
- meet 距离补偿 `meetMsPerPx` 已有，避免极近仍「拖沓」  
- exit 力度已与飞入/拖速挂钩，勿再叠一层无意义缩放爆炸  

### 4.2 抽牌 / 洗回

| 项 | 现行方向 | 建议 |
|----|----------|------|
| 抽：先移再翻 | 已钉 | **保持**；与发牌「到位再翻」同家族 |
| 洗回 stagger | ~40ms 族 | **保持**；N 大加速逻辑可与发牌共用工具函数 |
| recPause 50ms | 已钉 | **保持** |
| 消废自动补抽 | 有动画、禁瞬移 | **保持** |

**可优化：** 抽移与发牌共用 `flyCardLinear({from,to,ms,z})`，减少两套曲线不一致。

### 4.3 选中 / 拖拽

| 项 | 现行 | 建议 |
|----|------|------|
| 选中 1.1 + 轻晃 | 已钉 | **保持**；晃是「活」不是「晕」——幅度已克制 |
| 意图伙伴不放大 | 已钉 | **保持**（Staging：只强调一张） |
| 拖 1.16 / 倾角跟速 | 已钉 | **保持**；接近 Balatro 的「跟手重量」但更干净 |

**不要：** 给所有 free 牌加 idle 大幅浮动（谜题可读性优先于「赌场呼吸」）。

### 4.4 层级（CARD_Z）

```text
静止：seat / stock / waste / select 低带
动态：drag 8k · draw 8.5k · flip 8.6k · recycle 8.7k · match 9k+
```

**优化纪律：** 任何新动画先声明「飞中带 / 落座带」，禁止临时 `z=99999` 不回收。

## 5. 从外源「该学什么 / 不学什么」

### 5.1 动画师生存手册

| 学 | 用在 |
|----|------|
| Timing | 发牌/抽 120–160；exit 可更「重」 |
| Spacing / Ease | 禁止默认 linear |
| 极短 Anticipation | 发牌前库区微压 |
| Overlap / Stagger | 批量牌阶梯，勿纯串行拖死 |
| Follow-through | 落座后角与 scale 晚 1 拍收 |
| Staging | 同时最多一对 exit 抢视线 |

| 不学（默认） |
|--------------|
| 强 Squash 压扁牌面 |
| 表演级 &gt;500ms 当常用交互 |

### 5.2 Pyramid / Disney·TriPeaks

| 学 | 用在 |
|----|------|
| 按遮挡建台的发牌顺序 | §3.3 |
| 短 cascade 开场 | §3.2 |
| 从 Stock 出发的叙事 | 起点 |

| 不学 |
|------|
| 把开场做成主题 MV（Disney 糖可以有，我们默认无） |
| 照搬其具体 ms（公开无规格） |

### 5.3 Balatro

| 学 | 用在 |
|----|------|
| Juice **分层**（主/次/音/屏） | 架构思维：开场只开主层 |
| ±3° 有机角 | 与 flip tilt 一致 |
| 顺序演出教系统 | 我们用顺序教 **遮挡**，不是教倍率 |
| 重效果可关 | 未来设置预留 |

| **不学（默认）** |
|------------------|
| 常驻 CRT / 扫描线晃 |
| 分数量级震屏当发牌标配 |
| 手牌永久大浮动 |
| 每张计分弹簧戏 |

## 7. 状态机草图（全桌动画关系）

```text
                    ┌─────────────┐
                    │   idle      │←──────────────┐
                    └──────┬──────┘               │
              level enter  │                      │
                           ▼                      │
                    ┌─────────────┐               │
                    │ dealing     │ isBusy        │
                    │ (cascade)   │               │
                    └──────┬──────┘               │
                           │ last settle          │
                           ▼                      │
                    ┌─────────────┐               │
         ┌─────────►│ playing     │◄─────┐        │
         │          └──────┬──────┘      │        │
         │     pick/drag   │             │        │
         │                 ▼             │        │
         │          meet / drag          │        │
         │                 │             │        │
         │                 ▼             │        │
         │          match exit ──────────┤        │
         │          + flip free          │        │
         │          (exit 不占 busy)     │        │
         │                 │             │        │
         │          draw / recycle ──────┘        │
         │          (占 busy)                     │
         │                                        │
         └──── 下一操作 ──────────────────────────┘
```

**读法：**  
- **dealing / draw / recycle / meet**：占 busy  
- **exit / flip**：尽量不锁全局；单卡锁翻中  
- 新发牌状态应 **显式插入** level enter，不要 silently 瞬铺

## 8. 验收清单（可打勾）

### 8.1 发牌（新）

- [ ] 从 Stock/屏底飞出，不是 fade 出现  
- [ ] 底层先于表层（或已拍板的顺序）  
- [ ] 单张 120–160ms，整体约 1s 级  
- [ ] ease-in-out，无上抛弧  
- [ ] 飞行不穿帮；落稳 z 正确  
- [ ] 发牌中无法乱点；结束后立刻可操作  
- [ ] 无默认震屏/粒子盖牌  
- [ ] 20+ 张时不拖过 1.5–2s（有加速）  

### 8.2 既有（回归）

- [ ] 点消直线飞入 + 同 exit  
- [ ] 拖消 skipMeet  
- [ ] 上抛始可操作；翻中禁拖  
- [ ] 翻 free：breath 1.3 + 小倾角  
- [ ] 抽：先移后翻；洗回 stagger  
- [ ] 自动补抽有动画  
- [ ] pivot 全绕牌心  

### 8.3 观感主观（3 人 10 秒）

- [ ] 「像发牌」而不是「刷出来」  
- [ ] 「利落」而不是「等动画」  
- [ ] 「干净」而不是「花」  
- [ ] 遮挡关系一眼能懂

## 10. 一页纸结论

| 问题 | 答案 |
|------|------|
| 现在动画总调性？ | **短、直、绕牌心、动高静低、结果可抢先操作** |
| 最大缺口？ | **进关谜题发牌 cascade** |
| 怎么补？ | 复用飞入 120–160 + stagger 40–50 + 深度序 + busy |
| 向谁对齐？ | 手册 timing · Pyramid 顺序 · 自有 14；**不**对齐 Balatro 震屏秀 |
| 什么叫做好？ | &lt;~1.2s 铺完、遮挡正确、落座可玩、仍然干净 |
