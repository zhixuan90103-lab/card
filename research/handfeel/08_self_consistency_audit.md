## 3. 不自洽与张力（按严重度）

### 3.1 🔴 代码 · match 路径与 08 S3 / E06

**应然（08 S3 · 03 E06）：**

```text
freeBefore 采样
→ match commit
→ sync(skip=pair [, holdBack=toFlip])
→ 仅 HUD（勿全量 sync 冲 pair）
→ flyAway [busy]
→ onDone: flipToFace(toFlip) [!busy] → full sync
```

**实然（`main.ts` doTapCard / 拖匹配成功）：**

```text
match
→ sync(skip=pair)
→ refresh()          // ⚠️ 全量 cards.sync + HUD
→ flyAway → onDone: refresh()
// 无 freeBefore · 无 holdBack · 无 flipToFace
```

| 问题 | 影响 |
|------|------|
| flyAway **前** `refresh()` 再 sync | 可能冲掉 skip 意图 / 新 free 瞬时亮面（无 L4） |
| **无 E06** | 规格 P0 与代码 🔴；顿悟「掀开」弱 |
| 点消与拖消路径对称 | ✅ 两处同样「不完整」——至少一致地欠账 |

**判定：** 规格自洽；**规格↔代码不自洽**。关闭方式 = 轨 I（非再检索）。
