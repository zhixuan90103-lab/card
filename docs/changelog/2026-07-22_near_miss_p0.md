## 2. 实现（P0）

### 2.1 稀缺

| 层 | 范围 | 常量 |
|----|------|------|
| **硬门槛** | 每锁 match-key ∈ **[2, 4]** | `KEY_SCARCITY_HARD_LO/HI` |
| **偏好** | 每锁 ∈ **[3, 4]**（near-miss） | `KEY_SCARCITY_LO/HI` |

- `enforceLockKeyScarcity` 仍用硬门槛 2～4（保证能发出局）  
- `dealLevel01` **评分优先** 3～4  

### 2.2 前半进度

| API | 作用 |
|-----|------|
| `probeGreedyProgress` | 贪心路径：stall 前配对数、stall 时桌面张数、是否可清 |
| `passEarlyProgress` | stall 前至少 **3 对**（`EARLY_MIN_MATCHES_BEFORE_STALL`） |

### 2.3 发局选优（轻量，控耗时）

```text
base: 无跨锁埋钥 + 稀缺 2～4 + 桌面钥匙率
必须: passClearGreedy（hard）

得分（仅可清局）:
  4 = 稀缺 3～4 且 前半进度   → 立刻采用
  3 = 稀缺 3～4
  2 = 前半进度
  1 = 仅可清

流程:
  找到首个可清后，再 polish ≤12 次取更高分
  hard 禁 density 不可清；extreme 最后可密度兜底
```
