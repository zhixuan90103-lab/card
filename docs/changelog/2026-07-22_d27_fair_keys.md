## 2. 决策条文（写入 `04`）

**D27 公平钥匙几何**

1. **禁止跨锁埋钥**：任一锁堆内（除锁顶）不得出现 **另一把锁** 的 match-key。  
2. **钥匙独立可挖位**：优先非锁 L1 mid + 桌面挖开链；chain **不再**用埋钥表达顺序。  
3. **chain 语义收窄**：多锁节奏 / 并立标签，非「B 钥压在 A 下」。  
4. **两钥互消**：合法难度（玩家误用），须仍存在 **至少一条不依赖误用的可清路径**（hard 用贪心过滤保证）。  
5. **H1b 落实**：hard **禁止**不可清 density 兜底；找不到可清则扩 attempt / throw。  
6. **extreme**：优先可清；扩 attempt 后仍可密度兜底（最后手段），且 **仍过 D27 埋钥检查**。

未改：D22 同点同色、D24 稀缺 2–4、D23 难度档、D10 清桌胜。

### 生成伪代码（钥匙）

```text
for each lock i:
  place key at freeL1[i].mid   // 独立 C/D 位
if chain:
  extra keys on further freeL1 mids/bots  // 旁路，仍非锁下
// NEVER: lockGroup[A].mid = lockRank[B]
// when filling lock stacks: ban all lockRanks
```

### 发局过滤

```text
passKeyScarcity(2..4)
&& passNoCrossLockKeyBurial
&& passKeyOnBoard(minRatio)
&& (hard ⇒ passClearGreedy 必须 true
    extreme ⇒ 优先 clear，否则 densityOk 兜底)
```

## 6. 玩家向说明（可进 HUD 教学）

- 锁被盖住时，**同色同点**在别处挖出来再配。  
- **不要把两张同色同点钥匙先互消**——可能开不了锁。  
- 若提示卡住：撤销 / 重开同 seed / 新局。  
- 不应再出现「必须先开 A 才能摸到 B 钥匙」的堆内埋伏。
