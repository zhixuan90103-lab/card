## 3. 牌与布局数据（建议约定，便于实现）

### 3.1 点数
- 使用 rank：`A,2,3,…,10,J,Q,K`（13 种）或子集  
- 配对只比 rank，**花色仅表现**  

### 3.2 一关牌张数约束（可解性）
- 每个出现的 rank，在本关总张数建议为 **偶数**（否则易死局或依赖抽光逻辑错误）  
- 抽牌区 + 谜题区 = 本关全部牌  

### 3.3 布局字段（草拟，非最终 schema）

```text
Level:
  id
  cards[]:
    id, rank, suit?
    layer: 0..N          # 0=底层
    cell 或 x,y,w,h      # 占位；底层可强制对齐 4×5 格
  stock[]: rank 列表     # 抽牌区从顶到底或从底到顶约定一种
```

### 3.4 遮挡
- 牌 A 遮挡牌 B：`layer(A)>layer(B)` 且 矩形相交面积 > 阈值
