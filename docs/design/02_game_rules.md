# 02 · 游戏规则（现行版）

**版本：** v0.3  
**更新：** 2026-07-21  
**重要更正：** 谜题区 = **4×5 底层 + 向上分层遮盖**（类《羊了个羊》），**不是**经典三角 Pyramid Solitaire。  
**v0.3：** 补「有步 / 软卡 / 真死」分层；统一「清谜题区」用语。

---

## 0. 共识对齐（先读）

| 说法 | 含义 |
|------|------|
| ❌ 旧误解 | 经典纸牌三角塔（上尖下宽、1+2+3+…行） |
| ✅ 现行共识 | **底：4 列 × 5 行格子上的牌**；其上再 **一层层盖牌**；上层挡住下层；越往上越少，**最顶可到 1 张且翻开** |
| 体感参照 | 《羊了个羊》式 **层叠遮挡 + 配对消除**（我们是 **同点两张** 配对，不是三消进槽） |
| 题材 | 纸牌（点数/花色是皮） |
| 不是 | 接龙搭龙 / FreeCell / 经典金字塔和为 13 |

---

## 1. 目标与胜负

| 项目 | 规则 |
|------|------|
| **胜利** | **清除谜题区（层叠牌阵）全部牌**（D10） |
| **抽牌区 / 抽出叠** | **工具区**（D10b / D20）：帮桌上单点；过关允许残留，清桌后自动回收；**库牌不可点**，只能抽到抽出叠再配 |
| **平行剥** | 禁止「两组同顶、同次顶」廉价展开（D19 / R15） |

> Level01 实现与配点细节：`docs/design/15_level_rank_design.md` · 会话总整理：`docs/changelog/2026-07-22_level01_session_summary.md`
| **卡住 / 软死亡** | 无任何合法配对 → 提示 + **撤销 / 重开本关** |
| **计时** | 无默认倒计时 |
| **洗回** | 抽牌区空 → 抽出叠洗回抽牌区（无限） |

## 2. 牌面布局

```text
┌──────────────────────────────────────────────┐
│  抽牌区（Stock）         抽出叠（Waste）         │
│  [████] 背面             [顶张 可点]             │
│                          [下层抽出 不可点]        │
├──────────────────────────────────────────────┤
│  谜题区（层叠牌阵 · 类羊了个羊）                  │
│                                              │
│      顶层 … 可能只有 1 张（翻开）                  │
│         \  |  /                              │
│      中层：盖在底层部分格子/交叉上                 │
│                                              │
│      底层：4 × 5  背扣或按关卡翻开                 │
│      ■ ■ ■ ■ ■                               │
│      ■ ■ ■ ■ ■                               │
│      ■ ■ ■ ■ ■                               │
│      ■ ■ ■ ■ ■                               │
└──────────────────────────────────────────────┘
```

| 区域 | 作用 |
|------|------|
| **谜题区** | 4×5 底 + 上层遮盖；主玩法 |
| **抽牌区** | 未抽出的牌 |
| **抽出叠** | 已抽的牌叠放；**仅顶张可点** |
| **列间接龙** | **无** |
| **底部「一横排亮牌当塔底」的三角塔** | **不是本方案** |

## 3. 谜题区：4×5 底 + 分层遮盖

### 3.1 底层
- **固定网格：横向 4 × 纵向 5**（共 **20 个底格**）。  
- 每格上可有牌（开局按关卡放置）。  
- 被上层盖住时不可点；完全未被盖住时可点（并应翻开可见点数）。

### 3.2 上层如何「盖」
- 在底层（及更下层）之上继续放牌，形成 **第 2 层、第 3 层…**  
- 上层牌盖住下方一格或多格的重叠区域（实现上用 **包围盒/占位重叠** 判定，类羊了个羊）。  
- 层数与每层张数由关卡定；**越往上张数越少，最顶可以是 1 张且翻开**。

### 3.3 可选条件（能被点击）
一张牌可选，当且仅当：
1. **其上没有其它牌压住它**（无重叠遮挡），且  
2. **已翻开**（建议：变为无遮挡时自动翻开）。

### 3.4 与经典三角塔的区别（防再混）

| | 经典 Pyramid Solitaire | **我们（现行）** |
|--|----------------------|------------------|
| 形状 | 三角形行：1+2+…+7 | **矩形底 4×5 + 多层盖** |
| 参照 | 纸牌金字塔 | **羊了个羊层叠** |
| 消除 | 常为和 13 等 | **同点两张** |

## 4. 抽牌区与抽出叠

| 规则 | 内容 |
|------|------|
| **随时可抽** | 不论谜题区是否已有可配对，都可抽 |
| **抽出压栈** | 新牌盖在抽出叠顶上；**仅顶张可选** |
| **洗回** | 抽牌区空且抽出叠有牌 → 抽出叠洗回抽牌区 |

---

## 5. 配对与操作

### 消除
- 两张 **点数相同** → 可配对收走（花色不限，除非以后改）。

### 操作（全手动）
1. 点第一张 **可选牌** → 选中。  
2. 点第二张 **可选牌**：  
   - **同点** → 两张飞走消除；  
   - **不同点** → **改为选中第二张**。  
3. **不自动消除。**

### 可选来源
| 来源 | 条件 |
|------|------|
| 谜题区 | 无遮挡 + 已翻开 |
| 抽出叠 | **仅顶张** |

合法组合：谜题+谜题，或 谜题+抽出顶。

---

## 6. 一局流程

1. 开局：4×5 底层 + 上层遮盖布置好；抽牌区有牌；抽出叠空。  
2. 点两张可选同点 → 消除 → 可能露出下层牌。  
3. 需要时随时抽牌 → 压在抽出叠顶。  
4. 抽出顶与谜题自由同点可配。  
5. 抽光 → 洗回 → 继续。  
6. 谜题区无牌 → **胜利**。  
7. 无合法配对（见判定）→ 软提示撤销/重开。

---

## 7. 有步 / 软卡 / 真死（务必区分）

### 7.1 即时可配（不用再抽）

可点集合 S = 谜题区所有自由翻开牌 + 抽出叠顶（若有）。

- **有即时步：** S 的点数多重集中，某 rank 出现 ≥ 2（可点两下消掉）。

### 7.2 软卡（看起来没步，但再抽可能有）

- 无即时步，但 **F ∩ D ≠ ∅**  
  - F = 自由翻开牌的点数集合  
  - D = 抽牌区 ∪ 抽出叠 **全部**牌的点数集合  
- 含义：库里还有能配自由牌的点，只是还没到抽出叠顶 → **应继续抽 / 洗回**，不要弹死亡。

### 7.3 真死局

同时满足：

1. F 中没有任何 rank 个数 ≥ 2（自由牌之间也不能互配），且  
2. **F ∩ D = ∅**（库里也再没有任何能配当前自由牌的点）

→ 软提示 + **撤销 / 重开本关**（第一版不做扣命硬失败）。

### 7.4 关卡侧

- 做关应保证 **存在清场顺序**（可解）。  
- 每个 rank 在本关总张数建议为 **偶数**，降低结构性真死局。

## 8. 明确不做

| 不做 | 原因 |
|------|------|
| 经典三角 Pyramid 布局当主结构 | 与需求不符（已更正） |
| 列间接龙 | 变 Solitaire |
| 自动配对 | 玩家要求自选点击 |
| 默认 timer | 体验红线 |
| 第一版钥匙特殊规则 | 可选后期；内部可标关卡瓶颈 |

---

## 9. 参数（可调）

| 参数 | 现行 |
|------|------|
| 底层 | **4 × 5** |
| 上层层数/每层张数 | 关卡定；可收到顶 1 张 |
| 抽出叠 | 只露顶 |
| 洗回 | 无限 |
| 配对 | 同点 **2 张**（非羊了个羊三消进槽） |

---

## 10. 一句话

> **4×5 底层上叠多层遮盖（类羊了个羊）；点两张同点消；随时抽且抽出只露顶；抽空洗回；清场过关。不是三角纸牌塔，也不是接龙。**
