# 03 · 体验设计与创新原则

**更新：** 2026-07-21  

---

## 1. 体验主轴：顿悟

### 目标感受时间轴（单关）

```text
扫视（知道消同点、清谜题区）
  → 试探（点对 / 抽牌）
  → 卡点（自由牌配不出或路径紧）
  → 顿悟（先消哪对 / 抽到什么顶 / 掀开哪条线）
  → 兑现（连续可消、结构打开）
  → 清场过关
```

### 体验合同

| 要强化 | 要避免 |
|--------|--------|
| 「想通再消」 | 纯狂抽碰运气当唯一解法 |
| 消后掀开、连锁空间 | 消完无反馈 |
| 撤销敢试 | timer、无解骗关、关内硬广 |
| 手动点两下的确认感 | 自动消掉「我选的」 |

---

## 2. 创新原则（体验优先）

> **创新从体验出发；只换机制不换感受曲线，玩家会觉得没区别。**

| 原则 | 含义 |
|------|------|
| 先定主卖 E | 本产品主卖 **顿悟 E5** |
| 机制服务峰值 | 层叠遮挡、抽牌压栈都为「想顺序 / 想何时抽」服务 |
| 纸牌是皮 | 不靠「我们是纸牌」差异化，靠 **遮挡消除的决策节奏** |
| 干净是差异 | 默认可解、无 timer、软死亡——相对脏 Solitaire/Screw 克隆 |

### 不建议当创新的

- 默认 timer  
- 接龙规则换皮  
- 与 Goods 三消方法论硬套  
- Meta 膨胀（第一版）  

### 可探索（未立项）

- 关卡化「每关一个设计瓶颈」（内部标注，非特殊牌规则）  
- 抽出叠压栈带来的「现在配还是再抽会压住」轻决策  
- 后续：锁钥皮 = 特殊同点对叙事（需单独设计案）

## 3. 用户不认可点（调研 → 本产品映射）

| 调研雷区 | 本产品对策 |
|----------|------------|
| H5 timer | 不做默认 timer |
| H12 假死局 / ads-or-lose | 关卡可解；死局软提示；不做强制广告过关 |
| H10 关内硬广 | 体验阶段禁止关内打断（商业另议） |
| 误操作惩罚过重 | 撤销；不同点改选中而非罚 |
| 像又一个接龙 | 明确无搭龙；叙事与教学说「层叠配对清场」 |
| 被当成羊了个羊换皮 | 强调 2 点直消、无进槽条、抽牌压栈决策 |

来源：`research/sorting-market/02` `04` `08`

## 4. 关卡设计提示（体验向）

1. 保证可解（或极高比例可解）。  
2. 至少有一段时间需要 **规划消序或抽牌时机**，而非全程自动有对。  
3. 消关键牌后应有 **可见露出**，形成兑现。  
4. 难度旋钮：塔深度、自由度、点数分布、抽牌区构成——**不换玩法词**。  
5. 验收：玩家是否说「要想一下」，而非「就是接龙/无脑抽」。

## 5. 原型验收话术

| 合格 | 不合格 |
|------|--------|
| 要想先消哪对 | 就是接龙 |
| 抽牌要算会不会压住 | 一直抽就行 |
| 消完会掀开 | 看不懂为什么点不了 |
