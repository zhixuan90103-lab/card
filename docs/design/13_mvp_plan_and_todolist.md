## 1. MVP 定义

### 1.1 要证明什么

| # | 命题 | 怎样算证明 |
|---|------|------------|
| P1 | 规则可玩且无歧义 | 外人能过 1～2 关且不把规则讲成接龙 |
| P2 | 遮挡点选正确 | 盖牌点不到；自由牌稳定可点 |
| P3 | 抽牌压栈有决策感 | 有人会说「先消还是先抽」 |
| P4 | 顿悟有空间 | ≥3 关需要想消序/抽序，不是纯狂抽 |
| P5 | 技术路径可走 | PC phone-frame +（目标）iPhone 15 可点 |

### 1.2 成功 / 失败（结束 MVP 时）

| 通过 | 失败 |
|------|------|
| 规则无争议、无必现卡死 | 频繁「点不了」说不清 |
| 有人主动说要想消序/抽牌 | 全员狂抽无脑过关 |
| 无人叫成接龙 | 被叫成接龙且改不回来 |
| D15–D17 全部满足 | 半路换 Three / 全屏桌面不当手机框 |

### 1.3 明确不做（MVP 外）

- 关卡生成器 / 千关  
- 锁钥特殊规则、进槽三消  
- 广告 / IAP / 排行 / 多语言全量  
- 精美全套美术、图集生产管线周  
- Three 主渲染、双引擎对比  
- 原生 App / Capacitor 定案  

### 1.4 技术硬约束（不可违反）

| ID | 约束 |
|----|------|
| D15 | **PixiJS 8.19.x** 单引擎；HUD 用 **DOM** |
| D16 | 逻辑坐标 **393×852**；PC **phone-frame letterbox** |
| D17 | 可点集 = **逻辑 isFree + AABB**（引擎不单独决定合法性） |
| — | **无默认 timer** |
| — | `core/` 纯 TS，可单测，零依赖 Pixi |

## 7. 风险与对策

| 风险 | 对策 |
|------|------|
| 遮挡阈值扯皮 | 先固定阈值写进 level 注释；O08 仍可后调 |

| 关卡无解 | rank 偶数；M0 只保 1 关可清 |
| 想做成接龙 | HUD/教学文案禁用「搭上去」；见 `07` |
| 范围膨胀 | 未过 M0 不做生成器/美术周；**D7 美术可选且不得阻塞 M1** |

| Pixi 不熟 | 参考 Yaran v8；先 Events + 官方 performance tips |
