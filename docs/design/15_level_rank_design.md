## 1. 玩家问题（现象）

> 相同位置展开后，下方数字依旧相同。

典型体验：消掉某槽组顶后，立刻露出**同点**的下一张 → 像「假展开 / 白消」，顿悟弱、也像做关偷懒。

根因（旧 Level01）：

1. **顺序灌点** `RANK_CYCLE` 连续切片，与空间无关  
2. **教学覆盖** 只改 L2 组顶（如改成 8），组底仍可能是 cycle 留下的 **8** → 同组顶底同点  
3. 未约束「同组内点数互异」

## 2. 行业 / 同类做法（调研摘要）

| 来源 | 做法 | 对本项目的启示 |
|------|------|----------------|
| **Mahjong Solitaire 可解生成** | 反向落子：只在「当前 free 位」放一对同图，保证至少一路可清 | 配对消可用 **pair-first / reverse-deal**；先保证 parity 与可消路径 |
| **Match-2 三消关卡** | 先定对子多重集，再布局；避免「一揭开又是同一对」的廉价感 | 同槽不要叠同一 rank 的相邻两张 |
| **Disney / 多层牌堆配对** | 开局 free 上有明确可点对；库牌补 parity；堆内错点 | 开局 teach pair + stock 偶数补齐 |
| **项目既有文档** | rank 全局偶数；默认可解信任 H12 | 全局偶数硬约束 |

结论：配点应是 **独立系统**，与几何搭建分离；几何定槽，配点满足约束。

## 3. 配点不变量（Level 作者必守）

| ID | 规则 | 级别 |
|----|------|------|
| **R1 全局偶数** | 谜题 + stock 中每个 rank 张数为偶数 | 硬 |
| **R2 同组互异** | 同一 group 内任意两张 rank **不得相同**（展开后下方必变点） | 硬 |
| **R3 开局可点对** | 开局 free（L2 组顶 ± 自动翻开的 waste）中至少存在一对同 rank | 硬（教学关） |
| **R4 对子分散** | 同一对 rank 尽量不全部埋在同一局部（优先跨组/跨层） | 软 |
| **R5 教学对** | 可指定若干 free 顶为 teach pairs（如 7-7、8-8、9-9） | 可选 |
| **R6 可解路径** | 理想：反向落子可清；MVP：R1+R2+R3 + 人工/抽测可清 | 目标 |
| **R7 stock 精简** | 抽牌区 = 奇点补齐 + **至多 2 组** L1 抽牌伙伴对；**禁止 pad 到 16**；通常 ≤14 张 | 硬（Level01） |
| **R8 stock 低重复** | 抽牌区内同一 rank **≤2**；种类尽量分散，避免库里刷同点为难玩家 | 硬（Level01） |
| **R9 开局少对** | 开局 free **恰好一对**可消，无三同点抉择 | 硬（教学关） |
| **R10 挖掘链** | 消对后新 free 应能与**场上已有 free** 再成对，而非全靠抽牌 | 硬（L2 手写链） |
| **R11 少狂抽** | 开局 waste **不要**与 free 顶同点；进度以桌面链为主 | 硬（Level01） |
| **R12 软锁** | 至少 1 张「锁」：压住有价值下方；**外观与普通牌相同** | 硬 |
| **R13 钥匙** | 至少 1 把同点钥匙在可规划路径；简单关 **≥2** 把 | 硬 |
| **R14 软难度** | 不解锁也可通关（更难）；解锁则变顺 | 硬 |
| **R15 禁平行剥** | 任意两组**组顶相同**时，其**次顶**不得相同（跨层）。杜绝「两堆 5 同时消完下面都是 Q，再消又是同点」的廉价平行展开 | 硬（Level01） |

实现：`level01Deal.ts` 填点自上而下 + `depthBannedRanks` + `repairParallelPeels`（按顶点分桶、多轮收敛、终检）。

## 8. 锁与钥匙 + 重开随机（Level01）

**不单独做皮肤。** 实现：`level01Deal.ts` + `canFullyClear` 验收。

| 项 | 规则 |
|----|------|
| 几何 | **固定满槽** L0 5×4×3 + L1 4×3×3 + L2 3×2×2（**不挖空、不挪坐标**） |
| 重开 | **新 seed**：点数、开局对落在哪两个 L2 顶、锁数/锁位/钥匙/拓扑随机 |
| 锁位 | 仅 **L1 组顶**；开局 L2 free **无锁** |
| 拓扑 | `independent`（钥匙在可达区）或 `chain`（锁 i 压住锁 i+1 的钥匙） |
| 禁止 | 钥匙环、自锁（DAG） |
| 开局 | 恰好一对可消 |
| stock | **精简工具区**（R7）：奇点补齐 + 至多 2 组 L1 伙伴对；禁 pad-16；清桌后残留自动回收 |
| 胜利 | **仅清谜题区**（D10 / D10b）；库/抽出叠不挡胜利 |
| 平行剥 | **R15 / D19**：同顶 ⇒ 次顶互异 |

**会话总整理：** `docs/changelog/2026-07-22_level01_session_summary.md`

`insightNote` 可含 JSON meta（seed / lockIds / topology）供调试。

## 4. 推荐生成流程（本项目）

```text
1. 几何：按 L0/L1/L2 建槽（id/layer/tier/rect），不写 rank
2. 建多重集 bag：总张数 = 谜题 + stock 目标；全部成对
3. 先锁定 teach / 开局 free 顶点数（从 bag 取出）
4. 按组填充：每组 size 张，从 bag 取 **互不相同** 的 rank
5. 剩余进 stock；再跑一遍 R1 校验（应恒成立）
6. 单测：R1、R2、开局 free 有 pair
```

实现：`src/data/rankDesign.ts` + `buildLevel01()`。

## 5. 难度旋钮（后续关）

| 旋钮 | 易 | 难 |
|------|----|----|
| 开局 free 对数 | 2～3 对 | 0～1 对，依赖抽牌 |
| 同 rank 跨层距离 | 近（易连消） | 远 |
| stock 长度 | 短 | 长，洗回多 |
| 埋深（组内张数） | 2 | 3～4 |

## 6. 验收清单

- [ ] 任意 group 内 rank 全不同  
- [ ] rank 全局偶数  
- [ ] 开局至少一对可点（或 waste 自动翻后可配）  
- [ ] 无「揭开仍同点」的同槽相邻  
- [ ] 手玩一局无明显假死（软卡可抽）
