## 0. 设计目标（Invariant）

本规范不是补丁列表，而是**构造保证**下列不变式：

| ID | 不变式 |
|----|--------|
| I1 | 被上层**组**压住的牌，**不得**亮面、**不得**可点 |
| I2 | 同组内只有**组顶**可能 free；组中/组底在组顶未消前恒背面 |
| I3 | 同板层级（同 tier）的**不同组**互不作为「层」互压；遮挡只通过 **footprint** 判定 |
| I4 | 局部揭开：压住本牌的上层消掉后即可翻，**不**要求整层清空 |
| I5 | Free ⇔ 亮面；非 Free ⇔ 背面（渲染不得另起一套逻辑） |

消除规则（同点配对）见 `02`；本文件只管 **摆放 + 谁可点/可翻**。

## 1. 术语

| 术语 | 定义 |
|------|------|
| **组 (group)** | 同一锚点上的一叠牌。id 前缀相同，如 `d00_0/1/2` → 组 `d00` |
| **组顶** | 组内 `layer` 最大的存活牌 |
| **tier** | 板层级 L0=0, L1=1, L2=2… 表示搭建层次 |
| **layer** | 组内细 z（底 0、中 1、顶 2 相对组内；跨层用绝对 layer 段） |
| **footprint** | 组内**所有存活牌矩形的并集**（轴对齐包围盒） |
| **Free** | `isFree === true`：可点且必须亮面 |

## 3. 组内结构（上漏边）

同槽上漏边，张数可按层配置：

| 层 | 每组 | 组内 id | 组顶 |
|----|------|---------|------|
| L0 / L1 | **3** | `_0` `_1` `_2` | `_2` |
| L2 | **2** | `_0` `_1` | `_1` |

```text
L0/L1 三张：
  [A 背]  _0  y = topY - 18
  [B 背]  _1  y = topY - 9
  [C   ]  _2  y = topY       = 组顶

L2 两张：
  [A 背]  _0  y = topY - 9
  [B   ]  _1  y = topY       = 组顶
```

- 禁止同组坐标完全重合。  
- 仅组顶在 free 时亮面。

## 4. 板层搭建（十字缝递推）

### 4.1 规则

| 层 | 锚点 | Level01 | 每组 | layer | tier |
|----|------|---------|------|-------|------|
| L0 | 格心 | 5×4 = 20 | 3 | 0–2 | 0 |
| L1 | L0 十字缝 | 4×3 = 12 | 3 | 3–5 | 1 |
| L2 | L1 十字缝 | 3×2 = 6 | **2** | 6–7 | 2 |

```text
十字缝中心 = 下层锚点 (col,row) 与 (col+1,row+1) 的中点
```

### 4.2 同层

- 同一 tier 的组形成**半格错位平铺**，组顶之间 **不得** 用 layer 互压（同层 topLayer 相同）。  
- **禁止** 同层再叠横缝+竖缝双网格导致互压。

## 5. 可点性 / 翻面（核心设计 · 组遮挡模型）

### 5.1 为什么用「组」而不是「单卡 layer」

单卡比较 layer 时：

- 组 A 的**中/底** layer **小于** 邻组 B 的**组顶** layer  
- 若 B 组顶中心压在 A 牌堆下，A 的中/底**无法**在 layer 意义上盖住 B  
- 结果：B **错误亮面可点**（玩家可见 bug）

这不是调参问题，是**遮挡单元选错**。遮挡单元必须是 **整组 footprint**。

### 5.2 判定算法（规范 · v1.1）

```text
isFree(C):
  if C 非存活谜题牌 → false
  // 同组
  if 存在同组 O 且 O.layer > C.layer 且 rectsOverlap(O, C) → false
  // 其它组 G
  for each group G ≠ C.group:
    foot = union(rects of alive cards in G)
    top  = max-layer card in G
    if G.tier < C.tier → continue
    if G.tier > C.tier:
      if rectsOverlap(foot, C) → false     // 更高板层：任意重叠
      continue
    // 同 tier
    if G.topLayer > C.layer:
      if rectsOverlap(foot, C) → false     // 组顶更高：任意重叠
      continue
    if G.topLayer < C.layer → continue
    // 同高：仅中心落入 foot 才挡（不对称，防 free 顶互锁）
    if G.topId != C.id and center(C) ∈ foot → false
  return true
```

| 高度关系 | 几何条件 |
|----------|----------|
| 更高 tier / 更高 topLayer | **任意 ≥1px 重叠** 即挡 |
| 同 tier 且 topLayer 相等 | **C 中心 ∈ G.footprint** 才挡 |

> **禁止**用 `coverThreshold`（比例重叠，如 0.12）做「更高层」free 判定。  
> 上层只剩中/底时，对角邻接下层往往只剩 **2%～8%** 角部重叠——比例阈值会误判 free，  
> 表现为「被盖住的牌却翻开」。更高层：**任意可见遮挡 ⇒ 不可 free**（I1）。

`coverThreshold` 仅保留给软几何/旧 `isCovering` 测试，**不参与** `isFree`。

### 5.3 推论（设计保证）

| 场景 | 结果 |
|------|------|
| L2 任意存活牌与 L1 有 ≥1px 重叠 | L1 **不翻** |
| L2 只剩中/底，角部压住边缘 L1（重叠 &lt;12%） | L1 **仍不翻**（v1.1 根因） |
| 中心压在邻组 footprint 下（同高） | **不 free** |
| 两 free 顶边缘轻碰、中心互不落入 | 可同时 free |
| 上层整组消灭 | 其下可翻 |

### 5.4 Free ⇔ 亮面

渲染**唯一**依据 `isFree`；禁止渲染层自判。

## 9. 明确否决

| 否决 | 原因 |
|------|------|
| 整层 tier 锁死（有任意 L2 则 L1 全不翻） | 局部已空仍不翻，体验错 |
| 仅单卡 layer 跨组遮挡 | 组中/底挡不住邻组顶 → 盖住却亮 |
| 同层双网格互压 | 破坏「一层平铺」 |
| 渲染自算亮背 | 与逻辑漂移 |
